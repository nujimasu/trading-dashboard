  Trading Dashboard — 開発記録
  作成日: 2026年3月29日
================================================================

----------------------------------------------------------------
  背景・目的
----------------------------------------------------------------

従来のトレーディング分析は Claude Code のスキル（52個）と
エージェント（4体）によるMarkdown/JSON出力のみで運用していた。
全体像の把握が困難なため、ローカルWebダッシュボードを構築し、
市場環境から銘柄選定・個別分析を一画面で確認できるようにした。


================================================================
  完成した機能（7セクション）
================================================================

  ① 市場ヘルス
     全スクリーニング対象銘柄のうち「Stage 2アップトレンド」比率を
     全体スコア・セクター別・テーマ別（AI、半導体、核融合等）で表示。

  ② 経済指標
     FMP APIから取得した経済カレンダーを、ポジティブ/ネガティブ/中立の
     影響別カラーで一覧表示。

  ③ ビッグニュース
     直近の市場ニュースを②と同形式で表示。
     影響セクター・関連銘柄をタグで付与。

  ④ 週次推奨銘柄
     スクリーニングパイプライン（Stage 1〜6）で選出した候補を
     複合スコア順に一覧表示。クリックで詳細パネルを展開し、
     テクニカル指標・エントリー根拠・ファンダメンタル・
     キャンドルスティックチャートを確認できる。

  ⑤ 本日のエントリー
     週次推奨銘柄の当日の価格・RR・ブレイクアウト状況・
     出来高を反映した日次調整リスト。
     ENTRY_NOW / WAIT / WATCH / PASSED の4段階で判定。

  ⑥ 銘柄検索
     任意のティッカーを入力して即時分析。
     yfinanceからリアルタイムデータを取得し、
     買い / 様子見 / 見送り の判定・理由・チャートを表示。

  ⑦ 判断基準・説明（📖）
     買い・様子見・見送りの条件、スクリーニングファネルの解説、
     各指標の意味、リスク管理ルールをまとめた説明ページ。


================================================================
  技術スタック
================================================================

  Backend  : Python 3 + FastAPI + uvicorn
  Frontend : Vanilla HTML/CSS/JavaScript（ビルドステップ不要）
             Tailwind CSS (CDN)
             lightweight-charts v4.1.3（TradingView OSS）
  Database : SQLite（data/trading.db）
  価格データ: yfinance（無料・無制限）
  ファンダ  : FMP API（Financial Modeling Prep 無料プラン）

================================================================
  スクリーニングパイプライン設計
================================================================

  目的：約600銘柄を10〜20銘柄に絞り込む多段ファネル。
  FMP APIの消費を最小化するため、無料のyfinanceを最大活用。

  Stage 1  ティッカー取得    FMPが403なら静的リスト374銘柄を使用
  Stage 2  価格データ取得    yfinance一括DL（100銘柄バッチ）FMPコスト: 0
  Stage 3a Stage2アップトレンド 終値 > SMA50 > SMA200        FMPコスト: 0
  Stage 3b RSI 40〜70       過熱・モメンタムなしを除外       FMPコスト: 0
  Stage 3c MACD > Signal    上昇モメンタム継続確認           FMPコスト: 0
  Stage 3d 52週高値から15%以内 ブレイクアウト近接銘柄のみ    FMPコスト: 0
  Stage 3e 出来高収縮 or レンジ縮小  VCPプレチェック          FMPコスト: 0
  Stage 4  RR計算           RR < 1.5 を無条件除外           FMPコスト: 0
  Stage 5  ファンダメンタル  サバイバーのみFMP呼び出し       FMPコスト: 〜1〜2コール/銘柄
  Stage 6  複合スコアリング  Tier 1/2判定・weekly_picks生成  FMPコスト: 0


================================================================
  判定ロジック
================================================================

  買い推奨（Tier 1）
    ・RR ≥ 2.0
    ・Stage 2アップトレンド
    ・上記フィルタ全通過
    ・フルサイズエントリー可

  様子見（Tier 2）
    ・RR ≥ 1.5（2.0未満）
    ・Stage 2アップトレンド
    ・ハーフサイズエントリー

  見送り
    ・RR < 1.5（例外なし）
    ・Stage 2アップトレンド崩れ
    ・RSI範囲外（40未満 or 70超）
    ・いずれか1つでも条件未達なら機械的にスキップ

  ※勝率はフィルタ条件に含まない（RRが1.5以上あれば判断基準として十分）

  複合スコア（0〜100）の内訳:
    テクニカル指標: 40%
    RR品質:        30%
    VCPスコア:     20%
    ファンダ:      10%


================================================================
  RR計算の仕様（analyze_rr.pyのロジックを継承・改良）
================================================================

  エントリー価格: 直近終値
  ストップ価格:   過去20日間の最安値を基準に以下の制約を適用
                  ・最大 エントリー − (ATR × 2.0) まで（広がりすぎ防止）
                  ・最小 エントリー − (ATR × 1.0) まで（狭すぎ防止）← 改良点
                    ※ 改良前は直近安値がエントリー近辺の場合にRRが
                      100倍超になる異常値が発生していたため修正
  目標価格:       過去100日間の最高値（直近高値以上のデータがない場合はATR×3）
  ATR:            14日間の平均真の値動き幅


================================================================
  FMP無料プランで使えるエンドポイント（判明分）
================================================================

  ✅ /profile/{ticker}              セクター・時価総額・企業概要
  ✅ /income-statement/{ticker}     EPS・売上成長率
  ❌ /stock/list                    全銘柄一覧（403 → 静的リストで代替）
  ❌ /earnings-surprises/{ticker}   決算サプライズ（403 → スキップ）
  ❌ /economic-calendar             経済カレンダー（403 → スキップ）
  ❌ /stock_news                    市場ニュース（403 → スキップ）

  403エラーは画面に表示せず自動スキップする仕組みを実装済み。

================================================================
  起動コマンドまとめ
================================================================

  # 初回セットアップ
  pip3 install -r requirements.txt

  # フルパイプライン実行（週1回・月曜朝）
  python3 pipeline/run_pipeline.py

  # 日次軽量更新（毎平日）
  python3 pipeline/run_pipeline.py --daily-only

  # ダッシュボード起動
  python3 run.py
  → ブラウザで http://localhost:8000 を開く
